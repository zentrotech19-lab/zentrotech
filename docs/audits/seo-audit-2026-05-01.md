# SEO Audit — ZentroTECH (2026-05-01)

Auditor: senior technical SEO consultant
Source: `c:\Antigravity\TEMP projects\Zentro\zentrotech\` (Next.js 16, React 19)
Methodology: live curl of every route on `http://localhost:3030` (the project's actual dev port — the `localhost:3000` server referenced in the brief is a different project, `techvity-web-v4`, see Critical Issue #0), source review of `app/`, `lib/seo.ts`, `lib/constants.ts`, `components/seo/*`, and the eight insight + eight case-study MDX files.

## Summary

Overall SEO health: **5.5 / 10**. The technical foundation is mostly correct — every route returns 200, Next.js metadata API is wired, an Organization schema exists, `sitemap.ts` produces the right 32 URLs, `robots.ts` is sane, and `<title>`/`<meta description>` lengths are largely on-target. But three categories of failure compound: (1) **broken assets** referenced by every social/JSON-LD payload (`/og/default.png` and `/logo.png` both 404, see Critical Issue #1) which makes every share preview fail and the Organization schema invalid; (2) **city-page non-differentiation** — the Bangalore, Dubai, and UAE landing pages share three of four sections, ~225 words each, zero local references (no Karnataka GST regs, no DIFC, no DPDP Act, no Arabic content despite the spec calling for it), and the UAE page even injects `LocalBusinessSchema city="Dubai"` (a copy-paste bug); (3) **thin/missing content** — half the MDX files are <250 words, the showcase/services-detail bodies are 124-168 words, and there are no `Article`/`BreadcrumbList`/`FAQPage` schemas on insight posts. This is launch-blocking polish on a competent skeleton.

## Critical Issues (block launch)

### 0. The `localhost:3000` server in the brief is the wrong project
**Route:** all
**Evidence:** `netstat -ano | grep :3000` → PID 40160; `wmic process where ProcessId=40160 get CommandLine` →  
`C:\Antigravity\OG WEBSITE\techvity-web-v4\node_modules\.bin\..\next\dist\bin\next start`. Curling `http://localhost:3000/sitemap.xml` returned `https://techvity.in` URLs (a laptop-rental business). I had to start the actual ZentroTECH dev server on the configured port (`package.json` → `"dev": "next dev -p 3030"`). **Fix:** update the auditing brief to reference port 3030 (or change the script to 3000), and stop the unrelated Techvity server before any Lighthouse / external scan, or those tools will inadvertently audit the wrong site.

### 1. Every OG image and the Organization logo are 404
**Routes:** all (every page references `https://zentrotech.ai/og/default.png`; root `OrganizationSchema` references `https://zentrotech.ai/logo.png`)
**Evidence:**
```
$ curl -o /dev/null -w "%{http_code}" http://localhost:3030/og/default.png  →  404
$ curl -o /dev/null -w "%{http_code}" http://localhost:3030/logo.png        →  404
$ ls public/  →  brand/  email-templates/  marketing/  *.svg   (no og/ dir, no logo.png)
```
Rendered HTML on `/services`:
`<meta property="og:image" content="https://zentrotech.ai/og/default.png"/>`
**Impact:** every LinkedIn / Twitter / Slack / WhatsApp share will show the broken-image placeholder; Google's rich-result test will reject the Organization schema for missing `logo` URL; AI crawlers (ChatGPT, Perplexity) that resolve `og:image` for previews will fail. Also `metadataBase` in `app/layout.tsx` is set, but the leaf metadata in `lib/seo.ts` returns absolute URLs anyway, so the breakage is real, not relative-URL fiction.
**Fix:** create `public/og/default.png` (1200×630) plus per-route OG images (`/og/services.png`, `/og/about.png`, `/og/work-{slug}.png`, `/og/insights-{slug}.png`), and `public/logo.png` (≥112×112 per Google's structured-data spec). Even better: switch to Next.js's `opengraph-image.tsx` route file so each route generates its own preview at build time.

### 2. UAE landing page injects the wrong LocalBusiness city
**Route:** `/ai-development-uae`
**Evidence:** `app/ai-development-uae/page.tsx` line 20: `<LocalBusinessSchema city="Dubai" />`. Schema lookup by `OFFICES.find(o.city.toLowerCase() === city.toLowerCase())` therefore emits `name: "ZentroTECH — Dubai"` on a UAE-targeted page. Confirmed in rendered HTML: `"@type":"ProfessionalService","name":"ZentroTECH — Dubai"` appears on the `/ai-development-uae` payload.
**Impact:** Google sees the Dubai office address+phone+geo on a page whose `<title>` and H1 are about the UAE more broadly. This dilutes both pages' local relevance and may even cause the UAE page to be deduplicated into the Dubai page in the local pack.
**Fix:** either pass `city="UAE"` and add a `UAE` (or "Abu Dhabi") record to `OFFICES`, or — cleaner — make the UAE page a country-level `Organization` + `serviceArea: AE` and drop the `ProfessionalService` schema entirely. The UAE page is a country page, not a city page.

### 3. Contact page has no metadata export → inherits root `<title>`/`<description>` and is missing canonical
**Route:** `/contact`
**Evidence:** rendered HTML on `/contact`:
```
<title>ZentroTECH — AI Consultancy | Agents, Automation &amp; AI-Powered Websites</title>
<meta name="description" content="ZentroTECH is a Bangalore-headquartered AI consultancy ..."/>
<!-- og:url is "https://zentrotech.ai" (HOME url), no canonical link, no og:image -->
```
Source: `app/(marketing)/contact/page.tsx` exports `"use client"` and *no* `metadata`. Because client components can't export `metadata`, the page silently inherits the root layout's defaults — title and description duplicate the home page (a textbook duplicate-content signal), and no canonical is emitted at all.
**Fix:** split this into a server `page.tsx` that exports `metadata = buildMetadata({ title, description, path: "/contact" })` and renders a `<ContactForm />` client component. Title suggestion: `"Contact — Talk to ZentroTECH"`; description: 140-160 chars including "Bangalore + Dubai" and a phone/email mention.

### 4. Home page is missing canonical, OG image, and most Twitter card tags
**Route:** `/`
**Evidence:** rendered HTML — `<link rel="canonical">` is **absent**; OG payload has only 6 tags (no `og:image`, no `og:image:width/height/alt`, no `og:type` is set in source though one ends up emitted); Twitter card has only 3 tags (no `twitter:image`).
**Cause:** `app/page.tsx` has no `metadata` export. The root `app/layout.tsx` defines `metadata` but does not include `alternates.canonical` or an `openGraph.images` array.
**Fix:** add `export const metadata = buildMetadata({ title: SITE.name, description: SITE.description, path: "/", ogImage: "/og/home.png" });` to `app/page.tsx`, and add `images: [{ url: "/og/default.png", width: 1200, height: 630, alt: SITE.name }]` to the root layout `openGraph` block as a sitewide fallback.

### 5. `OrganizationSchema` `logo` URL is broken (validation failure)
**Route:** `/`
**Evidence:** `components/seo/organization-schema.tsx` emits `logo: "${SITE.url}/logo.png"`. Combined with Critical #1, Google's Rich Results test will throw "logo URL is unreachable / not a recognized image format". `name`, `url`, `description`, `sameAs` are otherwise good.
**Fix:** ship `public/logo.png` (square, ≥112×112, ideally 512×512) and `public/logo.svg`. Update schema to include both `logo` and `image`.

## Important Issues (fix before promotion)

### 6. City landing pages are 90% identical — no local content moat
**Routes:** `/ai-consultancy-bangalore`, `/ai-agency-dubai`, `/ai-development-uae`
**Evidence:** all three pages render the same three sections (`TrustSignals`, `ServicesBento`, `CTASection`); body word counts are 225 / 232 / 226 (essentially identical); only the H1 + the 60-word intro paragraph + the badge differ. None of the three mentions any of: Karnataka DPDP Act, ELCITA, Whitefield/Koramangala/Indiranagar (Bangalore); DIFC, ADGM, UAE Personal Data Protection Law (Federal Decree 45/2021), Dubai Internet City, Free Zone vs Mainland, Tashkeel/Etisalat (Dubai/UAE). Without unique local proof, Google will treat these as doorway-style duplicates and either pick one to rank or ignore all three. Competitors like fractal.ai and geekyants.com win local queries with case studies / local clients / local team photos / Hindi or Arabic press mentions on these pages.
**Fix:** each city page needs ≥800 unique words covering (a) named local clients or sectors served, (b) local regulatory/compliance specifics, (c) timezone/working-day reality (Sunday-Thursday for UAE), (d) office address with embedded `<map>`, (e) 3-5 local FAQs (FAQPage schema), and (f) a unique testimonial with local attribution.

### 7. Multilanguage / Arabic readiness is zero despite Dubai/UAE focus
**Routes:** `/ai-agency-dubai`, `/ai-development-uae`, all
**Evidence:** `<html lang="en">` is hard-coded in `app/layout.tsx`; no `hreflang` link tags on any page; no Arabic content variant; no `dir="rtl"` provision; no `i18n` config in `next.config.ts`. The brief's "spec mentioned Arabic for Dubai market" is unimplemented. A case study at `/work/al-marwa-retail-bilingual-voice-agent` actually advertises "Arabic + English voice agent serving 80 stores across the UAE" — selling bilingual voice-AI from a monolingual website is an own goal.
**Fix:** stand up an `app/[locale]/...` route or App Router parallel-routes structure with `en` (default) + `ar` (RTL) for at minimum the home, services overview, Dubai, UAE, and contact routes. Emit `<link rel="alternate" hreflang="en" href="...">` + `hreflang="ar" href=".../ar/..."` + `hreflang="x-default"` on every translated page.

### 8. No Article / BlogPosting schema on insight detail pages
**Route:** `/insights/agentic-ai-2026` (and all 7 sibling posts)
**Evidence:** rendered HTML on `/insights/agentic-ai-2026` shows `JSON-LD blocks: 0`. Source `app/(marketing)/insights/[slug]/page.tsx` only sets metadata via `buildMetadata` and renders the body — no schema injection.
**Impact:** loses eligibility for Google's Top Stories / News / "How-to" rich results, loses the article card carousel, loses author/datePublished signals to LLM crawlers.
**Fix:** in the `[slug]` page, render a `<JsonLd data={{ "@context": "https://schema.org", "@type": "BlogPosting", headline, description, datePublished, dateModified, author, image, publisher: { "@type": "Organization", name, logo }, mainEntityOfPage }} />`. Also emit `BreadcrumbList` for `Home > Insights > <Title>`.

### 9. No BreadcrumbList schema or visible breadcrumbs anywhere
**Routes:** all nested routes (`/services/[slug]`, `/work/[slug]`, `/insights/[slug]`)
**Evidence:** zero `BreadcrumbList` JSON-LD blocks across all 13 fetched pages; no visible breadcrumb UI in any layout.
**Fix:** add a `<Breadcrumbs>` server component that renders both visible nav (`Home / Insights / The Agentic AI Tipping Point`) and a `BreadcrumbList` JSON-LD payload. Place it directly under the header.

### 10. No FAQPage schema on city/service pages, despite obvious FAQ opportunities
**Routes:** `/ai-consultancy-bangalore`, `/ai-agency-dubai`, `/ai-development-uae`, `/services/[slug]`
**Evidence:** zero `FAQPage` schemas; pages also have no FAQ section in the visible UI. Competitors win the "people also ask" results on queries like "AI consultancy cost Bangalore" and "AI agency Dubai pricing" with 6-10 inline Q&As.
**Fix:** add a 6-question FAQ block to each city/service page and wire a `FAQPage` schema.

### 11. Half the content is thin (<250 words)
**Files:**
| File | Words |
|---|---|
| `content/insights/why-bangalore-is-ai-capital.mdx` | 175 |
| `content/insights/dubai-ai-opportunity.mdx` | 200 |
| `content/insights/agentic-ai-2026.mdx` | 209 |
| `content/work/nimbus-ai-website.mdx` | 147 |
| `content/work/vertex-labs-agent-platform.mdx` | 159 |
| `content/work/acme-corp-automation.mdx` | 163 |

Plus the actual rendered page bodies (counted excluding header/footer/scripts) for `/services/ai-agents` = 124 words and `/showcase` = 147 words. The Acme case study even contains the literal text `> **Note:** Sample case study for demo purposes. Replace with real client work.` — which is a guaranteed Google-spam signal and will appear in the SERP snippet if not removed.
**Fix:** every leaf page should be ≥600-800 words for SEO viability; case studies should be ≥1000-1500 with quantified results, named clients, and real screenshots; insight posts ≥1200. Strip the placeholder note immediately. Two of the eight insight posts (`agentic-ai-2026` is one of them) are linked from the home page (`InsightsPreview`) — they're effectively the public face of "thought leadership" today.

### 12. Sitemap drops 5 case-study slugs (32 URLs vs the brief's "all 37")
**Route:** `/sitemap.xml`
**Evidence:** 32 `<url>` blocks present. The brief asks for 37. Counted from `lib/constants.ts` + content: 10 static + 6 services + 8 insights + 8 case studies = 32. So the sitemap matches what exists, but **the audit brief's expected count of 37 implies five case studies/insights are still planned**. Either the spec is stale or content is missing.
**Fix:** reconcile content backlog vs `CONTENT.md`. Also: the home URL is emitted as `https://zentrotech.ai` (no trailing slash) — fine — but every other URL also has no trailing slash; pick one canonical form and enforce a redirect (Next defaults to no-trailing-slash, so verify `next.config.ts` doesn't re-enable `trailingSlash: true`).

### 13. `<meta name="viewport">` blocks accessible zoom
**Route:** all
**Evidence:** every page renders `<meta name="viewport" content="width=device-width, initial-scale=1"/>`. Acceptable, but the prior build (the cached Techvity output that 3000 served) had `maximum-scale=5` which would have been a WCAG fail. Verify Next 16 isn't auto-injecting `maximum-scale=5` from any default; current output looks clean.
**Fix:** keep current viewport. No `user-scalable=no` or `maximum-scale<5` allowed.

### 14. No `theme-color`, `apple-touch-icon`, or `manifest.json`
**Route:** all
**Evidence:** extraction shows `THEME: MISSING`, `APPLETCH: 0`, `MANIFEST: 0` on every route.
**Fix:** add to `app/layout.tsx`:
```ts
export const viewport: Viewport = { themeColor: "#0a0a0f" };
```
plus `app/icon.tsx` / `app/apple-icon.tsx` (Next 16 Metadata file conventions) and `app/manifest.ts` for a basic `webmanifest`.

### 15. City pages are orphaned from sitewide navigation
**Routes:** `/ai-consultancy-bangalore`, `/ai-agency-dubai`, `/ai-development-uae`
**Evidence:** `NAV_LINKS` in `lib/constants.ts` lists Services / Showcase / Work / About / Insights / Contact only. The footer maps `NAV_LINKS` and shows offices as plain text, not links. So the only internal-link source for the three city pages is the sitemap.xml.
**Fix:** add a "Locations" footer column linking to all three city pages; cross-link Bangalore ↔ Dubai ↔ UAE in their bodies; drop "Bangalore", "Dubai", "UAE" inline links into the home `GlobeSection` and `About` page.

### 16. Sitewide `<title>` + `<description>` repetition reduces SERP CTR
**Routes:** /services, /services/ai-agents, /work, /work/[slug], /insights, /insights/[slug], /about, /showcase, /contact
**Evidence:**
- `/services/ai-agents` title is `"AI Agent Development · ZentroTECH"` — 33 chars, well under the 50-60 sweet spot, missing keyword modifiers ("Bangalore", "Dubai", "Production", "2026")
- `/work` title is `"Work — Case Studies · ZentroTECH"` — 32 chars, generic; no industry, region, or year keyword
- `/about` title `"About — Our 2050 Promise · ZentroTECH"` — cute but `2050 Promise` is a brand-internal phrase nobody searches; no `About ZentroTECH` literal which is the most-searched form
- `/work/acme-corp-automation` description is only 89 chars; should be 140-160
**Fix:** rewrite titles to `<page intent> | <keyword> | ZentroTECH`. Examples:
  - `/services/ai-agents` → `"AI Agent Development for Production | Bangalore & Dubai | ZentroTECH"` (66 chars)
  - `/work` → `"AI Case Studies — Agents, Automation, Websites | ZentroTECH"` (60 chars)
  - `/about` → `"About ZentroTECH — AI Consultancy in Bangalore & Dubai"` (54 chars)

## Nice-to-Haves (optional polish)

1. **Performance hints in `<head>`**: only one `<link rel="preload">` per route; consider `<link rel="preconnect" href="https://fonts.googleapis.com" crossorigin>` (skipped because `next/font` self-hosts, but worth verifying), and `dns-prefetch` for any 3rd-party analytics you add. Per the perf audit, JS first-load is the bigger problem — preload won't fix that.
2. **`og:image` per route, not a single default** — better CTR on social.
3. **`twitter:site` / `twitter:creator`** are missing on every route. Add `twitter: { ..., site: "@zentrotech", creator: "@zentrotech" }` in `lib/seo.ts`.
4. **Author schema** on insights — currently emits an `author` string in the (yet-to-be-added) Article schema. Use `{ "@type": "Person", "name": "...", "sameAs": [linkedin, twitter] }` once you have a real author roster.
5. **Image-loading strategy** — current `IMG: 0` count on every page reflects pure-CSS / SVG / icon-font visuals. Once real client logos and product screenshots ship, ensure first-fold images use `<Image priority />` and below-the-fold default to lazy.
6. **`changeFrequency`** in `sitemap.ts` is set to "monthly" for all static routes, "yearly" for case studies and insights. Insights should be `monthly` (Google does pay attention to this hint, even if loosely). Case studies `yearly` is fine.
7. **`ItemList` schema on `/work` and `/insights` index pages** — both currently emit zero JSON-LD. An `ItemList` of children improves grouping in SERPs.
8. **Footer mentions "Hosted on Vercel"** — fine for transparency, useless for SEO; consider replacing with a city-pages link block.
9. **`SOCIAL.twitter`** points to `https://twitter.com/zentrotech` — switch to `x.com` (the canonical X domain) or both via `sameAs` array.
10. **`SITE.phone` is `"+91 80 0000 0000"` placeholder** — appears in OFFICES Bangalore record too; make sure NAP (name-address-phone) is real and consistent across schema, footer, contact page, Google Business Profile.

## What's Working Well

- **`sitemap.ts` and `robots.ts`** are correctly implemented via Next.js Metadata Routes; the served XML matches the source and references `https://zentrotech.ai/sitemap.xml` correctly. Robots is permissive without leaking `/api/`.
- **`lib/seo.ts` `buildMetadata` helper** centralises canonical + OG + Twitter generation, so once the OG image asset and root canonical are fixed, every page that uses it benefits at once.
- **Title/description lengths** on `/services` (66/152), `/insights` (62/136), `/showcase` (43/126), `/ai-consultancy-bangalore` (53/155), `/ai-development-uae` (51/149) are within or near the ideal SERP ranges.
- **`<html lang="en">`** is set; **viewport** is correct; the dark-theme `class="dark"` is applied at the `<html>` level (good for paint flash avoidance).
- **All 13 audited routes return 200** with sub-second initial responses on the dev server.
- **JSON-LD that exists** is syntactically valid (parsed cleanly via `json.loads`); `Organization` on home, `Service` on `/services/[slug]`, two `ProfessionalService` blocks on `/contact` (one per office), and `ProfessionalService` on each city page.
- **One H1 per page**, semantic H2/H3 cascade, no `<h1>` duplication detected.
